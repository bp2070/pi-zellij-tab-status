# Releasing

## One-time setup

1. Create an npm access token that can publish this package.
2. Add the token as a GitHub Actions repository secret named `NPM_TOKEN`.
3. Make sure the package name on npm is the one you want to publish.

## Release flow

This repository publishes from Git tags that look like `v*`.

Typical release steps:

```bash
npm version patch   # or minor / major
git push origin main --follow-tags
```

That does three things:
- updates `package.json`
- creates a matching git tag like `v0.1.1`
- pushes the commit and tag so GitHub Actions can publish

## What the workflow does

`.github/workflows/release.yml` will:

1. run on tag pushes like `v0.1.1`
2. verify the tag matches `package.json`'s version
3. run `npm pack --dry-run`
4. run `npm publish --provenance --access public`
5. create a GitHub release with autogenerated notes

## Dry run

Before cutting a release, you can verify the package contents locally:

```bash
npm run pack:check
```
